# 010 Simple AI Specification

## 1. Purpose

This spec defines how Simple AI should treat unsupported and manual card effects.

It does not define AI implementation code, algorithms, APIs, or UI.

## 2. Core Rule

Simple AI must choose from legal actions generated by the Rule Engine.

Simple AI must not bypass rule validation or directly mutate GameState.

Controller and LegalActionGenerator boundaries are owned by [012-controller-and-legal-actions.spec.md](012-controller-and-legal-actions.spec.md).

## 3. Effect Tag Usage

Simple AI may use `effect_tags` as heuristic hints.

Examples:

* prefer `draw` when hand size is low
* prefer `live_support` when attempting a Live
* prefer `energy_boost` when resource constrained
* deprioritize `manual_complexity` in AI vs AI debug mode

Tags do not make an effect executable.

## 4. Unsupported and Manual Effects

For `unsupported`, `tagged_only`, and `manual_resolution` effects, Simple AI must not assume automatic effect resolution.

Default AI vs AI debug policy for `manual_resolution` effects is `skip_and_log`.

Under `skip_and_log`, Simple AI ignores the unresolved semantic effect, records that the manual effect was skipped, and continues only through legal Actions.

Approximation by `effect_tags` is allowed only in explicit experimental mode. Silent auto-resolution is forbidden.

AI vs AI debug mode may support additional explicit run policies:

* `tag_approximation_experimental`: approximate by tags only when explicitly enabled
* `exclude_manual_cards`: exclude manual-resolution cards from the automated card pool

The chosen policy must be included in simulation output assumptions.

## 5. Determinism

Simple AI decisions must be deterministic when a seed is fixed.

Any random tie-break should be reproducible and logged.
